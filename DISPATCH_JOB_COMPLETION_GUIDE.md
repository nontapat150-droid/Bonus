# ระบบจบงาน / ทำไม่สำเร็จ - เอกสารการใช้งาน

## 📋 ภาพรวมของระบบ

เพิ่มให้ระบบแจกจ่ายงาน สามารถบันทึกสถานะการจบงานและงานไม่สำเร็จได้ พร้อมติดตามจำนวนกรณีงานสำเร็จสำหรับการคำนวณในระบบน้ำมัน

## ✨ คุณสมบัติใหม่

### 1. ปุ่มจบงาน (Complete Job)
- 🟢 **สีเขียว** - บ่งบอกการจบงานสำเร็จ
- ปรากฏบนหน้า:
  - รายการงาน (ด้านซ้าย)
  - แผนที่ (ด้านขวา)
- **เงื่อนไข**: เฉพาะงานที่มีทีมจ่ายแล้ว (assigned) และยังไม่เสร็จ

### 2. ปุ่มงานไม่สำเร็จ (Incomplete Job)
- 🔴 **สีแดง** - บ่งบอกว่างานไม่สำเร็จ
- **ต้องระบุเหตุผล**: เมื่อกดปุ่มนี้ระบบจะให้ป้อนหมายเหตุ/เหตุผลว่าเพราะเหตุใด
- ปรากฏบนหน้า:
  - รายการงาน (ด้านซ้าย)
  - แผนที่ (ด้านขวา)

## 🎯 วิธีการใช้งาน

### ขั้นตอนที่ 1: จบงาน
1. ค้นหางานที่ต้องจบ
2. ถ้าเป็นรายการ: กดปุ่ม "จบงาน" (ด้านล่างของการ์ดงาน)
3. ถ้าเป็นแผนที่: กดปุ่ม "จบงาน" ในรายชื่อด้านขวา
4. ยืนยันข้อมูลงาน (ระบบแสดง access_no, ลูกค้า, ที่อยู่)
5. ระบบบันทึกการจบงาน ✅

### ขั้นตอนที่ 2: ระบุงานไม่สำเร็จ
1. ค้นหางานที่ไม่สำเร็จ
2. ถ้าเป็นรายการ: กดปุ่ม "ไม่สำเร็จ"
3. ถ้าเป็นแผนที่: กดปุ่ม "ไม่สำเร็จ" ในรายชื่อด้านขวา
4. **กรอกเหตุผล** ในช่องหมายเหตุ เช่น:
   - "ลูกค้าไม่อยู่"
   - "สินค้าไม่พร้อม"
   - "ต้องเข้ามาครั้งใหม่"
5. กดปุ่ม "ยืนยัน"
6. ระบบบันทึกการไม่สำเร็จ ✅

## 🔗 การอินทิเกรชันกับระบบน้ำมัน

### เก็บข้อมูลของงานที่สำเร็จ

ทุกครั้งที่ช่างจบงาน ระบบจะบันทึกข้อมูลลงตาราง `job_logs`:

| ฟิลด์ | ความหมาย | ตัวอย่าง |
|------|----------|---------|
| job_id | ID ของงาน | 123 |
| tech_id | ID ของช่าง (สมาชิกทีม) | 5 |
| status | สถานะ: 'completed' หรือ 'failed' | completed |
| timestamp | เวลาที่บันทึก | 2026-05-30 14:30:00 |
| remark | หมายเหตุ (สำหรับ failed) | ลูกค้าไม่อยู่ |

## 💡 กรณีใช้งานระบบน้ำมัน

### 1. คำนวณ "กรณีงานต่อวัน"
```sql
-- ดึงจำนวนงานสำเร็จ ต่อช่าง ต่อวัน
SELECT tech_id, DATE(timestamp) as work_date, 
       COUNT(*) as completed_jobs 
FROM job_logs 
WHERE status = 'completed' 
GROUP BY tech_id, DATE(timestamp);
```

### 2. คำนวณประสิทธิภาพ (ค่าใช้น้ำมันต่องาน)
```sql
-- เปรียบเทียบจำนวนงาน กับ ปริมาณน้ำมัน
SELECT jl.tech_id, DATE(jl.timestamp) as date, 
       COUNT(jl.id) as completed_jobs,
       SUM(oil.liters) as total_liters,
       (SUM(oil.liters) / COUNT(jl.id)) as liters_per_job
FROM job_logs jl
LEFT JOIN oil_records oil ON jl.tech_id = oil.tech_id 
  AND DATE(jl.timestamp) = DATE(oil.date_recorded)
WHERE jl.status = 'completed'
GROUP BY jl.tech_id, DATE(jl.timestamp);
```

### 3. จัดสรรบันทึกน้ำมันตามจำนวนงาน
ในระบบน้ำมัน สามารถใช้ข้อมูลจำนวนงานเพื่อ:
- **ตรวจสอบความถูกต้อง**: ถ้าจำนวนงานน้อย แต่ปริมาณน้ำมันมาก = ผิดปกติ
- **คำนวณค่าใช้**: ค่าใช้น้ำมัน / จำนวนงาน = ค่าต่องาน
- **วิเคราะห์ประสิทธิภาพ**: วัดความมีประสิทธิภาพของแต่ละช่างตามความสัมพันธ์งาน:น้ำมัน

## ⚠️ ข้อควรรู้

1. **ต้องมีทีมจ่าย**: ปุ่มจบงาน/ไม่สำเร็จ จะปรากฏเฉพาะงานที่มีการจ่ายให้ทีมแล้ว
2. **บังคับเขียนเหตุผล**: หากงานไม่สำเร็จต้องระบุเหตุผล (บังคับ)
3. **ไม่สามารถแก้ไขซ้ำ**: งานที่ถูกบันทึกสแตตัสแล้ว ปุ่มจะหายไป
4. **ดูประวัติ**: ใช้หน้า "System History" เพื่อดูประวัติการเปลี่ยนแปลงทั้งหมด

## 🐛 การแก้ไขปัญหา

### ปัญหา: ปุ่มไม่ปรากฏ
- ตรวจสอบว่างานมีการจ่ายให้ทีมแล้ว (ดู column ทีม)
- ตรวจสอบว่าสถานะไม่เป็น "เสร็จแล้ว" หรือ "ไม่สำเร็จ" แล้ว

### ปัญหา: Error เมื่อจบงาน
- ตรวจสอบการเชื่อมต่ออินเทอร์เน็ต
- ตรวจสอบในคอนโซล Developer Tools (F12) เพื่อดูข้อผิดพลาด
- รีเฟรชหน้าและลองใหม่

### ปัญหา: ข้อมูลไม่อัปเดตในระบบน้ำมัน
- ตรวจสอบว่า `job_logs` table ได้ถูกสร้างแล้ว
- รัน migration: `sql/migration_add_job_logs.sql`
- ตรวจสอบ API: `api/dispatch/get_job_stats.php`
